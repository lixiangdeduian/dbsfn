# 任务说明

在 `database/sql/routines` 中已有多个存储过程/游标，希望在项目中提供可视化调用入口。

## 目标

- 顶栏新增「游标/过程」按钮，点击进入新的游标/过程展示页面。
- 新页面以卡片形式列出可用的存储过程/游标，显示名称/用途/参数提示。
- 点击卡片可发起调用，请预填合法的可用的参数，调用后给出成功/失败反馈和结果展示。

## 后端要求

- 暴露接口封装对 `database/sql/routines` 中过程/游标的调用，参数需校验和防注入（沿用现有 `sanitizeIdentifier` 等模式），保持现有 API 约定风格。
- 如需新增脚本或权限，更新 `database/sql` 相关文件并确保执行顺序兼容。

## 验收

- 在 UI 中点击「游标/过程」→ 进入展示页面 → 选择卡片 → 能成功调用对应过程并看到结果/提示。
- 现有功能不受影响；文档和配置（若有新增 env/权限）已同步更新。
